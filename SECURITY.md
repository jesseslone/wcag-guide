# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| 0.1.x   | Yes       |

## Reporting a Vulnerability

If you discover a security vulnerability in WCAG Guide, please report it
responsibly. **Do not open a public GitHub Issue for security problems.**

Please report vulnerabilities through **GitHub Security Advisories** — use the
"Report a vulnerability" button on the Security tab of this repository.

Please include:

- A description of the vulnerability and its potential impact.
- Steps to reproduce or a proof of concept, if possible.
- The version(s) affected.

You can expect an initial acknowledgment within 72 hours. We will work with you
to understand the issue and coordinate a fix before any public disclosure.

## Scope

This policy covers the WCAG Guide application itself: the API server, worker,
CLI, MCP server, and web dashboard. Vulnerabilities found on third-party sites
being scanned by the tool are outside the scope of this policy and should be
reported to the respective site owners.
